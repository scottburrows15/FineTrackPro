I am building a fines management app for a rugby club. I want you to implement an automatic bank transfer verification system using Open Banking. Please generate working code in Node.js with Express and PostgreSQL.

Requirements
Data model (Postgres tables)

teams (id, name, currency, ob_provider, ob_account_id, ob_consent_id, bank details, reference prefix)

players (id, team_id, full_name, nickname, email, status)

fines (id, team_id, player_id, amount_minor, currency, status, created_at, paid_at, payment_method, payment_reference, notes, deleted_at)

payment_intents (id, team_id, player_id, total_minor, currency, reference, status, created_at, expires_at, bank_details_snapshot, matched_transaction_id)

payment_intent_fines (payment_intent_id, fine_id)

transactions (id, team_id, provider_txn_id, amount_minor, currency, direction, booking_date, value_date, narrative, payer_name, iban_or_sortcode_acct, raw JSON, indexed_terms)

reconciliation_matches (id, transaction_id, payment_intent_id, confidence, match_reason, created_at)

audit_log (id, actor_id, team_id, action, entity_type, entity_id, metadata, created_at)

Endpoints

POST /api/payment-intents → create intent for one or more fines; generate unique reference; snapshot bank details.

GET /api/payment-intents/:id → view details.

POST /api/ob/consent/create → start bank connection via Open Banking (TrueLayer/Yapily/Plaid sandbox).

GET /api/ob/consent/callback → handle OAuth callback; store tokens.

POST /api/ob/sync → poll provider for new transactions; reconcile against payment intents.

GET /api/transactions → list matched/unmatched.

POST /api/reconcile/manual → admin links a transaction to an intent if auto-match fails.

Reconciliation logic

Normalise references (uppercase, strip spaces).

Match priority:

Exact reference match

Prefix + amount

Fuzzy match (e.g. Levenshtein)

If matched:

Mark payment_intent as settled.

Update linked fines → status = paid, set paid_at, payment_method=bank_transfer.

Write audit_log.

If multiple possible matches → mark as pending_verification.

Security

Store provider tokens encrypted.

Idempotent upsert of transactions by provider_txn_id.

Use parameterised SQL queries.

Extra

Add SQL migrations to create these tables.

Use a cron or worker (e.g. node-cron) to poll /api/ob/sync.

Include comments in the code so I can follow the flow.

Generate the following for me in the Replit project:

SQL migrations for all tables with indexes (especially on payment_intents.reference and transactions.provider_txn_id).

Express route handlers for the endpoints listed.

A background worker (node-cron) that polls the OB API and runs reconciliation.

Basic reconciliation pseudocode or helper functions.
